# Tiny Berlin Transit Display

**Vibecode project** — entirely AI-generated (Windsurf + OpenCode). No human wrote a single line of C++ or Python.

Mini Metro–style transit map on the **Waveshare ESP32-S3-Touch-AMOLED-1.8**. Fetches live VBB (Berlin transit) departures via a Python backend, renders a schematic map with animated vehicles. Touch to inspect stops/lines, two-finger zoom, auto sleep.

![hardware](https://www.waveshare.com/w/upload/thumb/1/18/ESP32-S3-Touch-AMOLED-1.8.jpg/400px-ESP32-S3-Touch-AMOLED-1.8.jpg)

## Hardware

| Component | |
|-----------| - |
| Board | Waveshare ESP32-S3-Touch-AMOLED-1.8 |
| Display | 368×448 CO5300 AMOLED via QSPI |
| Touch | CST816S self-capacitance (I²C) |
| PMU | AXP2101 |
| SD | microSD via SDMMC 1-bit |

Pin config in `pin_config.h`.

## Features

- Live VBB transit map (S-Bahn, U-Bahn, Tram, Bus) via HAFAS API
- Animated vehicles with progress interpolation
- Touch: tap stop/vehicle for info, two-finger zoom toggle
- BOOT button: short=brightness up, long=brightness down, double-click=screenshot via serial
- Config via SD card (`/.config/settings.json`)
- Auto light sleep when idle
- OTA updates

## Quick start

### Backend

```bash
cd backend
BRIDGE_TOKEN=change-me uv run uvicorn \
  transit_display_backend.main:app --host 0.0.0.0 --port 8000
```

Set `DEMO_MODE=1` env for synthetic test data (no VBB API needed).

### Build firmware

From repo root inside dev container:

```bash
bash compile.sh
```

Or use `task build` (Taskfile.yml).

### Flash to board (from host)

Find your ESP32 serial port first:

```bash
ls /dev/serial/by-id/*ESP32*  # or
ls /dev/serial/by-id/*JTAG*
```

Then flash:

```bash
python3 -m esptool \
  --chip esp32s3 \
  --port /dev/serial/by-id/REPLACE_WITH_YOUR_PORT \
  write-flash 0x0 \
  build/esp32.esp32.esp32s3/transit_display.ino.merged.bin
```

### SD card config

Place `/.config/settings.json` on the SD card:

```json
{
  "wifi": { "ssid": "YOUR_SSID", "password": "YOUR_PASSWORD" },
  "transit_display": {
    "profile": "home",
    "refresh_sec": 30,
    "stale_after_sec": 120,
    "sleep_timeout": 300,
    "bridge_url": "http://YOUR_SERVER_IP:8000",
    "bridge_token": "change-me",
    "brightness": 100
  }
}
```

All values are examples — change WiFi, bridge IP/token, and brightness for your setup.

The optional red dot on the map is configured in `backend/config/profiles.json` (`user_position` — set lat/lon or leave `null`).

## Vibecode origin

This project was built entirely through AI pair programming: architecture, implementation, debugging, and documentation all generated by LLMs (Big Pickle, GLM, chatGPT, Gemini). Human role was limited to testing on real hardware, providing feedback, and occasionally fixing AI hallucinations.

The code is not perfect — there are spartan error paths, minimal comments in places, and the C++ is "Arduino C++" not "modern C++". But it works reliably on the target hardware.
